# 交易 App 截图分析维度

## 用途

当用户提交东方财富、同花顺等交易 App 截图时，按以下维度提取结构化信息。

## 截图类型识别

先判断截图属于哪种类型：

| 类型 | 特征 | 提取重点 |
|------|------|---------|
| **持仓截图** | 显示股票列表、成本价、盈亏 | 持仓标的、仓位分布、盈亏情况 |
| **交割单/成交记录** | 显示买卖时间、价格、数量 | 具体操作记录、买卖时点 |
| **K线/走势图** | 价格图表 | 技术形态、关键价位 |
| **账户总览** | 总资产、收益率 | 整体业绩、资金规模 |
| **自选股列表** | 关注的票 | 关注标的、板块分布 |
| **个股详情** | 某只股票的详细信息 | 基本面、涨跌幅、量价 |

## A 股特殊规则

- **颜色语义**：红色 = 上涨/盈利，绿色 = 下跌/亏损（与欧美相反）
- **涨跌停标识**：+10%（或+20% 创业板/科创板）为涨停，-10%（或-20%）为跌停
- **ST 标识**：ST/\*ST 开头的为风险警示股
- **股票代码**：6 位数字（600xxx 沪市主板 / 000xxx 深市主板 / 300xxx 创业板 / 688xxx 科创板）

## 提取模板

### 持仓截图提取

```markdown
**持仓分析**：
- 总持仓数：{N} 只
- 持仓明细：
  | 代码 | 名称 | 成本价 | 现价 | 盈亏% | 仓位占比 |
  |------|------|-------|------|-------|---------|
  | {code} | {name} | {cost} | {price} | {pnl}% | {weight}% |
- 总盈亏：{total_pnl}
- 仓位集中度：{集中/分散}
- 板块分布：{sectors}
```

### 交割单提取

```markdown
**交易记录**：
- 标的：{code} {name}
- 操作：{买入/卖出}
- 时间：{datetime}
- 价格：{price}
- 数量：{quantity}
- 成交金额：{amount}
```

### 账户总览提取

```markdown
**账户状况**：
- 总资产：{total}
- 今日盈亏：{daily_pnl} ({daily_pnl_pct}%)
- 总盈亏：{total_pnl} ({total_pnl_pct}%)
- 持仓市值：{position_value}
- 可用资金：{available}
- 仓位比例：{position_pct}%
```

## 分析维度

提取完数据后，按以下维度分析：

1. **仓位管理评估**：是否符合用户 DNA Layer 0 的仓位规则？
2. **止损执行评估**：有没有亏损超过止损线还在持有的？
3. **板块集中度**：是否过度集中在单一板块？
4. **盈亏比**：赢的票赚了多少，亏的票亏了多少，比例合理吗？
5. **操作频率**：交易频率是否与用户声明的风格一致？

## 输出要求

- 先展示提取到的**事实**（数据）
- 再展示**分析**（对照 DNA 规则的评估）
- 最后**提议**（这些信息是否应该记录为经验？涉及哪个 DNA 层级？）
