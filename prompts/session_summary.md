# Session Summary — 会话记忆持久化

> 本 prompt 在每次悟道咨询会话结束时触发，生成 session summary 并写入本地存档。
> 下次会话开始时，加载最近 3 条 summary 作为上下文，实现记忆不中断。

## 触发条件

- 用户主动说"结束""下次聊""拜拜""退出悟道"等告别语
- 对话超过 15 轮时，主动询问是否生成本次摘要
- 模式 B 进化操作完成后

## 生成规则

阅读本次完整对话，生成以下结构的 summary：

```markdown
# Session Summary
- 日期：{YYYY-MM-DD HH:MM}
- 模式：{创建/进化/咨询/混合}
- 轮次：{对话总轮数}

## 本次概要
{用 2-3 句话概括本次对话的主题和走向}

## 情绪轨迹
{用户的情绪变化：焦虑→冷静 / 膨胀→清醒 / 沮丧→坚定 / ...}

## 新增数据
{列出本次新增的内容，如果有的话}
- 新经验条目：{#N - 简述}
- 自选股变动：{增/删了什么}
- 卖飞记录：{新增了什么}
- DNA 更新：{Layer X 第 N 条被修改/新增}
- 跨 skill 学习：{从 X skill 学了什么}
- 纠正：{第 N 次纠正，内容}

## 悟道分身表现
{本次使用了哪种语音模式：严格/鼓励/谦逊/分析}
{效果如何：用户是否被说服/鼓励到/冷静下来}

## 待关注事项
{未展开的话题、需要后续跟进的、用户可能想继续聊的}
```

## 存储位置

将生成的 summary 保存到：
```
${SKILL_DIR}/profile/sessions/{YYYYMMDD_HHMMSS}.md
```

## 加载规则（供下次会话使用）

下次悟道分身被激活时：
1. 读取 `${SKILL_DIR}/profile/sessions/` 目录下最新的 3 个 summary 文件
2. 将内容作为上下文注入，帮助悟道分身了解近期状态
3. 自然延续之前的对话状态，不做汇报式回顾

**注意**：不要主动说"上次我们聊了XXX"，除非用户问起。记忆应该是自然流露的——比如在鼓励模式中自然引用"上次你从-15%的回撤中走出来了"。

## 更新 meta.json

生成 summary 后，更新 `meta.json` 中的 `stats.sessions` 计数 +1。
