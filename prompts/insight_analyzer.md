# 雪球/论坛内容分析维度

## 用途

当用户提交雪球帖子、论坛讨论、博主文章或经验分享内容时，按以下维度提取有价值的交易洞察。

## 输入来源

- 雪球（xueqiu.com）帖子/讨论
- 东方财富股吧帖子
- 同花顺社区内容
- 微信公众号炒股文章
- 其他投资论坛内容
- 截图或粘贴的文字均可

## 提取维度

### 1. 核心论点

```markdown
**核心论点**：
- 作者观点：{一句话概括}
- 论点类型：{个股分析/板块分析/市场判断/交易策略/经验总结/心态分享}
- 结论方向：{看多/看空/中性/策略性}
```

### 2. 论据质量评估

```markdown
**论据质量**：
- 数据支撑：{有具体数据/仅定性描述/纯情绪输出}
- 逻辑链完整度：{完整/有缺环/跳跃性大}
- 信息时效性：{当日/近一周/过时}
- 独特视角：{有预期差/老生常谈/人云亦云}
```

评估等级：
- **A级**：有数据+逻辑完整+独特视角 → 值得深入学习
- **B级**：逻辑可参考但需验证 → 可作为参考
- **C级**：情绪输出或信息过时 → 仅供了解

### 3. 可操作的 Takeaway

```markdown
**可提取的交易洞察**：
1. {具体洞察}
   - 可操作性：{高/中/低}
   - 与用户 DNA 的匹配度：{高/中/低}
   - 建议归类：{DNA Layer X / 经验日志 / 仅了解}
```

### 4. 风险提示

```markdown
**需要注意的**：
- 作者可能的偏见/利益立场：{持仓方说多/空头方说空/中立分析}
- 幸存者偏差：{只展示成功案例？}
- 后视镜效应：{事后诸葛亮？}
```

## 与 DNA 的关联

分析完内容后，判断是否值得纳入悟道分身：

1. **A级内容 + 高匹配度**：建议写入 cross-skill-log.md 或 experience-log.md，可能升级到 DNA Layer 2/5
2. **B级内容**：记录在 experience-log.md 作为市场情报参考
3. **C级内容**：不建议记录，但如果用户坚持，以"市场噪音"标签记录

## 输出要求

- 区分**作者原文**和**AI 分析**
- 不自动接受作者观点为正确——交易观点需要验证
- 与用户 DNA 有冲突时，明确标注冲突点
- 如果内容质量很低（纯情绪、无逻辑），直接告知用户"这个参考价值不大"
