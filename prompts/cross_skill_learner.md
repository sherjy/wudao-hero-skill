# 跨 Skill 学习处理器

## 用途

当用户想从其他交易 persona skill（陈小群/北京炒家/李大霄/大曾子）中吸收特定观点或拒绝某些理念时使用。

这是悟道 skill 最独特的机制——无现有先例。

## 触发识别

以下表达触发跨 skill 学习模式：

**吸收（正向学习）**：
- "陈小群说的 X，我觉得有道理"
- "北京炒家的 X 方法我想学"
- "李大霄说的 X 我认同"
- "把 X skill 的 Y 观点加到我的悟道里"
- "我想学习 X 的 Y 思维"

**拒绝（反面教材）**：
- "大曾子的 X 教训，我要引以为戒"
- "陈小群的 X 不适合我"
- "不要学 X 的 Y"

## 处理流程

### Step 1：识别源 Skill 和目标内容

1. 从用户表述中识别源 Skill（使用 Skill 源映射表）
2. 识别用户想学习的具体内容
3. 判断是"吸收"还是"拒绝"

### Step 2：读取源 Skill

用 `Read` 工具读取源 Skill 的 SKILL.md，定位用户提到的内容：

- 在"核心心智模型"部分找对应模型
- 在"决策启发式"部分找对应规则
- 在"表达 DNA"部分找对应风格

### Step 3：提取完整内容

提取目标心智模型/规则的完整描述，包括：
- 一句话总结
- 核心逻辑
- 适用条件
- 局限性

### Step 4：转化

**关键步骤** — 不能照搬，必须转化为用户自己的版本：

1. **语境适配**：根据用户的资金量级、交易风格调整
   - 例：陈小群的"重仓怼龙头"→ 小资金用户版："精选1-2只龙头集中持有"
2. **声音转化**：用用户 DNA Layer 3 的表达风格重写
   - 不是"陈小群说..."，而是"我从情绪周期中学到..."
3. **冲突检测**：检查是否与用户现有 DNA 原则冲突
   - 冲突时标注 `[⚠️ 张力]`，让用户决定

### Step 5：预览确认

```
从 {source_skill} 学到的：

  原始观点：{original}
  
  转化为你的版本：{adapted}
  
  建议写入：DNA Layer {N}
  
  {如果有冲突}
  ⚠️ 注意：这与你现有的 Layer {X} 第{Y}条可能冲突：
  "{existing_rule}"
  如何处理？
    [A] 保留两者（标注张力）
    [B] 新观点替换旧规则
    [C] 放弃学习（现有规则更适合我）

确认整合？
```

### Step 6：写入

1. 备份当前版本（version_manager.py）
2. 追加到 `cross-skill-log.md`：

```markdown
### {日期} — 来源：{source_skill_name}（{正向学习/反面教材}）

**原始观点**：{original_text}
**我的理解**：{adapted_text}
**DNA 整合**：已写入 Layer {N} 第{M}条 / 未整合（仅记录）
**来源技能**：{skill_name}
```

3. 如果是原则级别（用户确认），merge 到 `trading-dna.md` 对应层级，标注 `[跨学:{skill_name}:{concept}]`
4. 更新对应的 `references/cross-skill-notes/from-{skill_slug}.md`
5. 重新生成 WUDAO-SELF.md
6. 更新 meta.json（cross_skill_learnings +1, source_skills_referenced 追加）

## 反面教材处理（大曾子特殊处理）

从大曾子学习时，采用"反向提取"：

- 大曾子的"全仓梭哈" → 用户的"永远分仓"
- 大曾子的"追涨杀跌" → 用户的"不追高，不恐慌割"
- 大曾子的"不止损死扛" → 用户的"铁律止损"

格式标注为 `[跨学:大曾子:反面教材:{具体概念}]`

## 注意事项

- 一次只学习一个概念，不要批量导入
- 每次学习都需要用户确认，不自动整合
- 来源标签是永久的——即使后来用户修改了这条规则，来源仍然保留
- 如果用户后来说"这个不适合我了"，通过纠正机制（correction_handler.md）移除
