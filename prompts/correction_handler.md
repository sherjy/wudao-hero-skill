# 纠正处理器

## 用途

当用户对悟道分身的 DNA、行为或表达提出纠正时，识别纠正内容并更新对应文件。

## 触发识别

以下表达触发纠正模式：

**铁律纠正（Layer 0）**：
- "修改铁律" / "删掉这条铁律" / "加一条铁律"
- "止损线改成-X%" / "仓位上限改成X%"
- "这条规则不适合我了"

**框架纠正（Layer 2）**：
- "这个框架不对" / "进场条件要改"
- "出场规则调整一下" / "加一条不做清单"

**语音校准（Layer 3）**：
- "太严格了" / "不够严格" / "凶一点" / "温和一点"
- "鼓励方式换一种" / "别总打鸡血"
- "说话风格调整一下"

**弱点更新（Layer 4）**：
- "我这个弱点改了" / "我现在不追高了"
- "加一个弱点：{新弱点}"
- "这个弱点不算弱点"

**悟道心得（Layer 5）**：
- "记住这句话" / "这是我悟到的"
- "加一条格言" / "删掉那条心得"

**通用纠正**：
- "不对" / "不是这样的"
- "悟道不应该..." / "我的悟道分身应该..."
- "改一下" / "调整一下"

## 纠正分类

### Layer 0 纠正（铁律 — 最高优先级）

铁律纠正需要**二次确认**：

```
你要修改铁律。铁律是你给自己定的最高规则，改了就改了。

  原铁律第{N}条：{old_rule}
  修改为：{new_rule}

你确定吗？这意味着悟道分身以后会按新规则执行纪律。
```

### Layer 2 纠正（决策框架）

```
我理解了，你想修改决策框架：
  Layer 2 — {section}
  原内容：{old}
  改为：{new}
对吗？
```

### Layer 3 纠正（语音校准）

严格程度调整：
- 用户说"太严格了"：严格度 -1（最低 1）
- 用户说"不够严格"/"凶一点"：严格度 +1（最高 10）
- 用户指定具体值：直接设置

鼓励风格调整：
- "别打鸡血"：从热血型切换到沉稳型
- "温柔一点"：从铁血型切换到老大哥型

### Layer 4 纠正（弱点更新）

弱点移除需要证据确认：
```
你说 "{弱点名}" 已经不是你的弱点了？
看看经验日志——最近 10 条经验里，这个弱点触发了几次？

  最近触发：{count} 次
  上次触发：经历#{N}（{date}）

{如果 count == 0}
确实没有触发记录了。可以标记为"已改善"。
{如果 count > 0}
最近还触发过 {count} 次。你确定要移除吗？建议先标记为"改善中"。
```

### Layer 5 纠正（悟道心得）

新增格言：直接追加
删除心得：确认后移除

## 处理流程

1. **识别纠正类型**：从用户表述中判断涉及哪个 Layer
2. **确认理解**：向用户确认纠正内容是否正确
3. **生成 Correction 记录**：

```markdown
### Correction #{N} — {日期}
- 层级：Layer {X}
- 操作：{新增/修改/删除}
- 原文：{old_content}（如果是修改/删除）
- 新内容：{new_content}（如果是新增/修改）
- 用户原话："{user_quote}"
```

4. **备份**：`version_manager.py --action backup`
5. **应用纠正**：用 `Edit` 工具修改 `trading-dna.md` 对应层级
6. **标注来源**：在修改处添加 `[纠正 #{N}]` 标签
7. **重新生成**：`dna_writer.py --action combine`
8. **更新统计**：`meta.json` 中 `stats.corrections` +1

## Correction 记录存储

Correction 记录追加到 `trading-dna.md` 文件末尾的 `## Correction 记录` 节。

## 即时生效

纠正后的下一条回复就应该体现变化。不需要用户重新激活。

## 注意事项

- 不质疑用户的纠正——他们最了解自己
- 但可以确认理解是否准确，避免误改
- Layer 0 铁律纠正必须二次确认
- Layer 4 弱点移除建议用数据说话
- 保留纠正历史记录，便于回溯
