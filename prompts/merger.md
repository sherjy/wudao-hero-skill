# 增量 Merge 逻辑

## 任务

当用户追加新的素材或经验时，将增量信息 merge 进现有的 profile 文件，不覆盖已有结论。

## 原则

1. **增量不覆盖**：新信息追加到已有内容后面，不替换已有结论
2. **冲突标注**：如果新信息与已有 DNA 原则矛盾，用 `[⚠️ 张力]` 标注，让用户决定
3. **强化标注**：如果新信息验证了已有原则，用 `[强化 ×N]` 更新计数
4. **证据升级**：如果新材料提供了更充分的证据，可以强化已有结论的置信度
5. **来源追踪**：每次 merge 都标注来源和日期

## 流程

### 1. 分析增量内容

将新素材按对应的 analyzer prompt 分析，输出：
- 新的经验条目（参考 experience_builder.md）
- 新的 DNA 影响（强化/新增/冲突）
- 与现有内容的一致性评估

### 2. Merge 策略

**经验日志（experience-log.md）**：
- 新条目 → 追加到文件末尾，序号递增
- 标签追加到全局标签统计

**Trading DNA（trading-dna.md）**：

| DNA 层级 | Merge 规则 |
|----------|-----------|
| Layer 0 铁律 | **不自动修改**。只有用户显式纠正才能变更。如果新经验与铁律冲突，标注 `[⚠️ 张力]` 让用户决定 |
| Layer 1 身份 | 几乎不变。除非用户明确说"我现在转型了" |
| Layer 2 决策框架 | 新的心智模型/规则可以追加。已有规则通过经验强化计数 |
| Layer 3 表达 DNA | 通常不变。除非用户纠正语音风格 |
| Layer 4 弱点档案 | 新经验触发弱点 → 更新触发次数。弱点改善 → 更新改善追踪 |
| Layer 5 悟道心得 | 新经验蒸馏出的高层原则 → 追加。跨 skill 学习 → 追加 |

**自选股（watchlist.md）**：
- 新票 → 追加行
- 已有票状态变更 → 更新状态列
- 卖出的票 → 状态改为"已卖出"，如果卖早了 → 同时追加到 regret-log.md

**跨 skill 学习日志（cross-skill-log.md）**：
- 新学习 → 追加条目
- 同一 skill 的多次学习 → 追加到同一 skill 区域

### 3. 冲突处理

当新经验与 DNA 原则产生冲突时：

```markdown
⚠️ 检测到张力：

新经验 #{N} 的教训是：{lesson}
但你现有的 DNA Layer {X} 第{Y}条说：{existing_rule}

这两者可能矛盾。你怎么看？
  [A] 保留原有规则（新经验是特例）
  [B] 修改规则（新经验揭示了规则的不足）
  [C] 两者都保留（标注为未解决张力）
```

### 4. 输出

用 `Edit` 工具追加内容到对应文件的对应章节，并在新内容前标注：

```markdown
<!-- [追加于 {日期}，来源：{来源类型}] -->
{新内容}
```

### 5. 后续操作

每次 merge 完成后：
1. 调用 `version_manager.py --action backup` 备份
2. 调用 `dna_writer.py --action combine` 重新生成 WUDAO-SELF.md
3. 更新 `meta.json` 中的对应 stats 计数

## 智慧蒸馏触发

当 experience-log.md 中新增条目累计达到 20 条的倍数（20, 40, 60...）时，自动触发一次智慧蒸馏：

1. 回顾最近 20 条经验
2. 统计标签频率，找出重复模式
3. 提取高频教训，候选为 DNA Layer 5 新原则
4. 询问用户确认后写入
